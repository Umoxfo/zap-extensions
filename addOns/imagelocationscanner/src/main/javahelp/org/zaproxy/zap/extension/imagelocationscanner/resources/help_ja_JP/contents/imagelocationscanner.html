画像位置情報・プライバシースキャナー

Webサイトの通常のセキュリティ評価中に、画像内のGPS位置情報やその他のプライバシー関連の露出を受動的にスキャンします。 Image Location and Privacy Scanner (ILS、画像位置情報・プライバシースキャナー) は、エンドユーザーがプロフィール画像を投稿し、自宅の位置を漏らす可能性がある状況（例: 出会い系サイトや子供向けチャットルーム）で役立ちます。

このトピックに関する詳細情報（OWASP組織のニュージャージー支部でのプレゼンテーションとして行われた実際のサイト監査に基づくホワイトペーパーを含む）は、https://www.veggiespam.com/ils/でご覧いただけます。

このソフトウェアは、画像をスキャンしてExifタグ、IPTCコード、カメラ独自タグ内のGPS情報を検出します。 その後、ILSはZAPのアラートリストに検出結果を情報メッセージとしてフラグ付けします。 位置情報の露出が文脈に基づいて真にセキュリティリスクとなるかどうかを判断するのは監査者の役割です。

WebブラウザをZAP経由でプロキシするように設定し、以下のサンプルサイトを閲覧してアラートが発生する様子を確認してください：

  • MetaData Extractorの SampleOutput ページ には良好な画像がいくつか含まれています。 (注: 一部のURLではGitHubセッションクッキーが必要です)

  • このプロのフォトグラファーは、フルサイズ (サムネイル以外) の写真の多くでExif & IPTCデータを活用しています: Raia.com

使用上の注意

  • ZAP 2.7.x以前では、ILSが機能するために画像スキャンを手動で有効にする必要があります: ツール → オプション → 表示 → 「HTTP リクエスト/レスポンス内の画像を処理する」
  • デフォルトでは、ZAPは履歴に画像を表示しませんが、ILSはこれらの画像を検出結果についてスキャンします。 アラートがトリガーされた場合、画像とそのアラートはアラートタブに表示されますが、履歴タブには表示されません。 アラートの有無にかかわらず履歴に画像を表示するには、上記の「HTTPで画像を処理する」を有効にしてください。
  • ツール → オプション → ネットワーク → グローバル除外 → 拡張子 - 画像（旧称：グローバル除外URL）を使用して画像処理を完全に無効にしている場合、ILSのようなパッシブ画像スキャナーは画像を認識できず、プライバシー問題を報告できません。
