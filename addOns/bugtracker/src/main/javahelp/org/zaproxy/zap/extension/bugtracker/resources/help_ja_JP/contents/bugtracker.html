バグトラッカー

バグトラッカーアドオンを使用すると、Webアプリケーションをスキャン中に検出されたアラートを、Bugzilla や GitHub などのバグトラッカーに課題 (Issue) やバグとして提出できます。

バグトラッカーの設定

ユーザーは、オプション -> バグトラッカー に移動し、使用するバグトラッカーを選択して、ユーザー名、パスワード/トークン、リポジトリURLなどの必要な情報を設定できます。

課題の提出

スキャン後、ユーザーは1つ以上のアラート (必要に応じてそのサブノードも) を選択し、コンテキストメニュー項目 -> 「バグトラッカーに課題を提出」を選択できます。
新しいダイアログが表示され、フィールドを編集したり、既存の設定から一部のフィールドを選択したりできます。 たとえば、GitHub を選択した場合、ユーザー名、パスワード、リポジトリURLが正しければ、これらの項目を入力する必要はありません。 「バグトラッカーの選択」ドロップダウンから、問題を報告するバグトラッカーを選択できます。
入力項目は、選択したバグトラッカーとその実装に応じて異なります。
必要な情報を入力したら、「保存」をクリックして問題を報告します。 報告が成功した場合はその旨が、失敗した場合は理由がメッセージとして表示されます。
