DOM XSS アクティブスキャンルール

DOM XSS脆弱性を検出するためのアクティブスキャンルールです。

ブラウザウィンドウを起動し、関連するすべてのDOM要素に攻撃ペイロードを送信します。
ブラウザウィンドウを起動するため、他の (ブラウザベースでない) ルールと比べて大幅に時間がかかります。

このバージョンはFirefox (デフォルト)、Chrome、Edgeをサポートしています。これらはGUIまたはヘッドレス (デフォルト) で実行できます。設定は ルール rules.domxss.browseridで変更できます。オプションの「ルール設定」パネルから、以下の値を使用します: firefox, firefox-headless, chrome, chrome-headless, edge, edge-headless.

攻撃強度としきい値

以下の攻撃強度がサポートされており、URLフラグメントとフォーム入力フィールドのインジェクションに使用される攻撃ペイロードの数に直接関連しています (例: http://example.com/index.html?foo=bar#injection)：
  • LOW: 1個の攻撃ペイロード
  • MEDIUM: 3個の攻撃ペイロード
  • HIGH: 6個の攻撃ペイロード
  • INSANE: 9個の攻撃ペイロード
スキャナーは、選択された強度に影響されないURL/クエリパラメーターのインジェクションも試行します。

このルールは、LOWアラートしきい値が使用されている場合を除き、ノードごとに1つのDOM XSS脆弱性のみを報告します。 LOWアラートしきい値の場合は、指定されたすべてのペイロードを試行し続けます。

除外

このルールは、起動したブラウザが以下によって除外されたURLにアクセスすることをブロックします：
  • グローバル除外
  • コンテキスト除外 (関連するコンテキストで呼び出された場合)
