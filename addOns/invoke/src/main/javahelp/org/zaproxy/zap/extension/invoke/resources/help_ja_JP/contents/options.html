オプション – アプリケーション画面

この画面では、起動可能な アプリケーション を設定します。
初期状態では登録されているアプリケーションはありません。使用したいアプリケーションをすべて追加してください。

表示名

ZAPでこのアプリケーションに使用される名前です。

コマンドのフルパス

起動するアプリケーションの完全なパスを指定します。
指定できるコマンドは1つのみです。複数のコマンドを一度に実行する場合や環境変数を設定する場合は、スクリプトまたはバッチファイルを作成し、それをZAPから起動してください。

パラメーター

ZAPがコマンドに渡すパラメーターです。
パラメータを渡すために以下のタグを使用できます：
     %url% 完全なURL。(例: http://localhost/test?a=b)
     %site% サイト。(例: http://localhost:8080/)
     %host% ホスト名。(例: localhost)
     %port% ポート。(例: 80)
     %cookie% リクエストヘッダーの最初のCookieフィールド (存在する場合)
     %postdata% 送信されたPOSTデータ (存在する場合)。すべての改行は "\n" に置換されます
     %msgid% APIからデータを取得するためのHTTPメッセージID (/JSON/core/view/message/?id=$msgid)。 常に存在するわけではなく、メッセージIDがない場合は-1を返します
     %header-{{ header }}% 名前によるリクエストヘッダー (存在する場合)。例: %header-user-agent%はUser-Agentヘッダーの値を返します
したがって、コマンドが '-u' を使用してURLを受け入れる場合、パラメータは次のように指定します: '-u %url%'

出力をキャプチャ

チェックすると、実行されたコマンドとそれが生成する出力が 出力タブに表示されます.
UIを持たないスクリプトやコマンドを実行する場合は、常にこのオプションを使用してください。
このオプションはトラブルシューティングにも使用できます。アプリケーションが期待どおりに実行されない場合は、実行されたコマンドをコピーしてコマンドラインプロンプトから試してください。

ノートに出力

If checked then any output will be appended to a ノート attached to the relevant 履歴タブ record.
Note that some nodes in the サイトタブ do not have associated 履歴タブ records - for those nodes this option will have no effect.
