SOAP アラート

以下のアラートは、SOAP アドオンによって発生します。

アラート参照 名前 説明 最新のコード
90026 アクションスプーフィング SOAP リクエストには、後にウェブアプリケーションによって実行される何らかの操作が含まれます。 この操作は、SOAP ボディの最初の子要素で見つけることができます。 しかし、HTTP が SOAP メッセージの転送に使用される場合、SOAP 標準では SOAPAction と呼ばれる追加の HTTP ヘッダー要素の使用が許可されます。 このヘッダー要素には、実行される操作の名前が含まれます。 これは、受信側のWebサービスに SOAP ボディに含まれる操作を、XML パースなしで通知することを目的としています。 この最適化は、攻撃者によって攻撃を仕掛けるために使用される可能性があります。なぜなら、特定のWebサービスフレームワークでは、SOAPAction ヘッダーフィールドに含まれる情報のみに基づいて実行される操作を決定するからです SOAPActionSpoofingActiveScanRule.java
90029 SOAP XML インジェクション 「XML インジェクション」では、攻撃者は SOAP メッセージ内のさまざまな XML タグを追加または操作しようとし、XML 構造を操作することを目的とします。 通常、成功した XML インジェクションは、制限されたまたは意図しない操作の実行を引き起こします。 実行された操作によっては、さまざまなセキュリティまたはビジネスコントロールが侵害される可能性があります。 SOAPXMLInjectionActiveScanRule.java
90030 WSDLファイル検出 このアラートは、パッシブスキャナーが WSDL ファイルを検出したときに発生します。 WSDLFilePassiveScanRule.java
