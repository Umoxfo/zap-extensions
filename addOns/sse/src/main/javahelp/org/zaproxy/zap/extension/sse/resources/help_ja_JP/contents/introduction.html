サーバー送信イベント

サーバー送信イベントは、WebアプリケーションやWebサイトで、単一のTCP接続上に一方向 (片方向) の通信チャネルを確立するために利用できます。

サーバーからブラウザクライアントへのプッシュ通知に利用できます。 サーバー送信イベントは、Ajax、HTTP Long Polling、またはCometの使用に対する代替手段となる場合があります。

ブラウザでは、EventSource API を介してサーバー送信イベントを使用できます。 このAPIはHTML5に属し、W3C (http://www.w3.org/TR/eventsource/) によって標準化されています。


ZAPは以下のことが可能です：

  • サーバー送信イベントをインターセプトし、表示する
サーバー送信イベントは、 「サーバー送信イベントタブ」内に表示されます.
